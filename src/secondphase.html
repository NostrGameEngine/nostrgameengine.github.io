A game engine and framework for building games and applications integrated with the Nostr ecosystem.

Second Phase

Summary of the first phase

During the initial development phase, the Nostr Game Engine was bootstrapped from the jMonkeyEngine codebase, enhanced, and integrated with several traditional and Nostr-powered features:
  • The original network stack was fully replaced with a peer-to-peer architecture powered by Nostr signaling , enabling decentralized, permissionless connectivity between players.
  • A modular component-based architecture was implemented, allowing developers to easily write reusable code and share it across projects.
  • Core Nostr functionality was implemented via nostr4j, a purpose-built high-performance and memory efficient client library developed from scratch.
  • A Nostr-based advertising system was created, both as an engine module and as a standalone library, allowing developers to integrate permissionless advertising into their games and applications.
  • Matchmaking and player profile management have been implemented over Nostr, leveraging its decentralized event model for identity persistence and social discovery without centralized servers.
  • A managed authentication subsystem was developed, capable of automatically authenticating players using NIP-07, NIP-46, or locally encrypted private keys.
Multiple upstream patches and extensions were applied to jMonkeyEngine, enhancing its overall performance, feature set, and Android support.
Furthermore, the entire engine was ported to WebGL2 and modern web technologies via TeaVM, enabling cross-compilation to JavaScript with near-native performance. This makes it possible to deploy NGE-based games directly to browsers, no installation required.

Finally, a comprehensive build and deployment Gradle template was integrated into the engine’s app template. This toolchain supports packaging for all target platforms and enables deployment to Nostr-based static websites through the third-party service nostrdeploy.

Overview of the second phase

The second phase will first focus on integrating some additional Nostr-based social features, expanding the engine’s capabilities and strengthening its multiplayer and community-driven components.

After that, development will concentrate on improving abstractions and developer accessibility. This stage aims to make the engine more intuitive for newcomers.
Video tutorials, documentation, and example projects will be created to showcase the engine’s features and support the development of Nostr-powered games and applications.

Then, the second phase will proceed by extending the engine to support Bitcoin-based micropayments, VR applications, and interoperability between games and experiences, allowing them to connect and integrate over Nostr.
