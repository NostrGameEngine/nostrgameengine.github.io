A game engine and framework for building games and applications integrated with the Nostr ecosystem.

Second Phase

Summary of the first phase

During the initial development phase, the Nostr Game Engine was bootstrapped from the jMonkeyEngine codebase.

The original network stack was fully replaced with a peer-to-peer architecture powered by Nostr signaling , enabling decentralized, permissionless connectivity between players.

Core Nostr functionality was implemented via nostr4j, a purpose-built client library developed from scratch. This library provides support for advanced Nostr capabilities, including NWC, Blossom, and other extensions.

A Nostr-based advertising system was also created—both as an engine module and as a standalone library—allowing developers to integrate permissionless ad networks directly into their games or related applications.

Matchmaking and player profile management have been implemented over Nostr, leveraging its decentralized event model for identity persistence and social discovery without centralized servers.

The engine now includes a managed authentication subsystem capable of automatically authenticating players using NIP-07, NIP-46, or locally encrypted private keys.

Multiple upstream patches and extensions were applied to jMonkeyEngine, enhancing its overall performance, feature set, and Android support.

Furthermore, the entire engine was ported to WebGL2 and modern web technologies via TeaVM, enabling cross-compilation to JavaScript with near-native performance. This makes it possible to deploy NGE-based games directly in browsers—no installation required.

Finally, a comprehensive build and deployment system Gradle template was integrated into the engine’s app template. This toolchain supports packaging for all target platforms and enables deployment to Nostr-based static websites through the third-party service nostrdeploy.


Overview of the second phase

The second phase will focus on integrating additional Nostr-based social features, expanding the engine’s capabilities and strengthening its multiplayer and community-driven components.

A subsequent stage will concentrate on improving abstractions and developer accessibility. This stage aims to make the engine more intuitive for newcomers. Video tutorials, documentation, and example projects will be created to showcase the engine’s features and support the development of Nostr-powered games and applications.

Later stages will extend the engine to support Bitcoin-based micropayments, VR applications, and interoperability between games and experiences, allowing them to connect and integrate over Nostr.
